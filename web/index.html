1. Profile by identity

UPS access API — confirms the identity exists. Profile → Edge copies the profile entity.identityMap into sendEvent (namespace ecid → ECID, same email id as UPS). That aligns Edge with the profile lookup; empty propositions usually mean wrong decision scopes / spoof URL for the site where experiences are published, or Edge errors — not a missing ECID.

Uses the get.decisions Direct Call identifier in §3 (not experience-decisioning-lab). Fetches UPS, then _satellite.track with UPS + eventType: 'get.decisions' + identityMap when parseable.

Streaming HTTP API (header.msgType: xdmEntityCreate): after UPS, POST the envelope to your Flow Service inlet URL (https://dcs.adobedc.net/collection/… from the base connection’s inletUrl). The proxy sends your IMS bearer + x-api-key + org + sandbox (authenticated inlet). Tune dataset, schema, and tenant path to match your XDM.

2. Experience Decisioning (REST)

Many orgs require x-schema-id for /offer-items (decision item schema). List decision items needs a schema URI unless your org allows the call without it. Item collections and selection strategies often work with None. REST calls require python proxy_server.py (repo root) — static servers have no /api/aep.

3. Content decisioning (Edge)

Loads Adobe Launch (development). For the main flow, use Profile → Edge decisions + render in section 1: Edge sendEvent runs only after UPS returns a profile. Email above is sent in identityMap on that call.

On real DSN, Launch sets scopes from the page; on localhost this mimics that URL so Edge can match the same channel config. The path must include the same site id as in web://dsn.adobe.com/…#… (e.g. apalmer-I018, not a truncated I01). If you enter scopes above, those win; leave scopes empty to use the two defaults.

Tags rule: Direct Call identifier experience-decisioning-lab — map Edge payloads into the page (channel DOM, etc.). Payload includes decisions / propositions from the Web SDK result when available.

Separate Tags rule: Direct Call identifier get.decisions — fired by Profile + Direct Call in §1 with UPS + eventType in the payload (your rule logs/sends XDM to AEP).

Debug: Edge without UPS check

Journey webhook feed (poll)

Best: repo proxy python proxy_server.py (default :8765, or PORT=8877 python proxy_server.py to match your static server port). Fallback: if /api/webhook-listener 404s, the lab tries the public listener URL directly in the browser when CORS allows.

Use an AJO custom action (or any step that can POST JSON) to the listener’s HTTPS URL only: https://webhooklistener-pscg5c4cja-uc.a.run.app (TLS end-to-end; Cloud Run shows x-forwarded-proto: https). This lab polls the index with GET https://… (proxy or browser fallback). The feed list only shows entries whose JSON has config.customName equal to Content Decision (on the root object or on any element of a top-level data[] array, typical AJO success shape). Map bodies to include decisions or propositions when possible for Preview + apply to demo DOM; other JSON still appears in the list as raw when it passes that filter. Webhook stores may contain PII.
