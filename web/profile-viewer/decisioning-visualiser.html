Method One
Item Priority
Assign a static numeric score to each offer. The engine always surfaces the highest-scoring eligible offer — no computation, no profile data required.
Live demo — drag sliders to set priority scores
A streaming platform is running three concurrent upgrade promotions. The marketing team sets priority scores to control which offer gets shown first to any eligible visitor.
An airline is promoting three ancillaries on the pre-trip website and app. Revenue management sets priority scores so the highest commercial-value bundle surfaces first to every eligible traveller.
A retailer is running three concurrent checkout and homepage promotions. Merchandising sets priority scores so the highest-margin or strategic offer surfaces first for every eligible shopper.
A bank is promoting three products on mobile and web — mortgage, card, and ISA. Marketing and risk set priority scores so the commercially aligned, eligible offer surfaces first for every customer in scope.
A telco is promoting three concurrent acquisition offers — unlimited mobile, gigabit fibre, and SMB connectivity. Marketing and wholesale set priority scores so the strategic, eligible offer surfaces first for every subscriber in scope.
An OEM and dealer group are promoting three concurrent programmes — new-car PCP, EV charger bundle, and inclusive service plan. Sales and aftersales set priority scores so the commercially aligned, eligible offer surfaces first for every contact in scope.

◆ Profile attributes — preferred genre, subscription tier, viewing frequency
◆ Item attributes — content genre, end date, price point
◆ Context data — channel, device type, time of day
◆ Logical operators — boosts, multipliers, conditionals
◆ Profile attributes — loyalty tier, home hub, trip purpose (business / leisure)
◆ Item attributes — ancillary type, route, cut-off hours before departure
◆ Context data — channel (web, app, kiosk), device, days until departure
◆ Logical operators — boosts, multipliers, conditionals
◆ Profile attributes — loyalty tier, price band, shopper segment (value / premium / member)
◆ Item attributes — category, margin band, hours until promo end
◆ Context data — channel (web, app, store), device, cart value, store pickup
◆ Logical operators — boosts, multipliers, conditionals
◆ Profile attributes — customer intent (everyday / wealth / SMB), risk band, product holdings
◆ Item attributes — product line, APR / fee, hours until rate-lock or offer end
◆ Context data — channel (branch, web, app), authenticated session, consent flags
◆ Logical operators — boosts, multipliers, eligibility rules
◆ Profile attributes — subscriber segment (mobile / home / SMB), tenure, usage & spend band
◆ Item attributes — product line, contract term, hours until offer or install window ends
◆ Context data — channel (app, web, retail, care), network type, roaming or in-market
◆ Logical operators — boosts, multipliers, eligibility & fair-use rules
◆ Profile attributes — buyer segment (showroom / EV / fleet), vehicle age, finance & service history
◆ Item attributes — programme type, stock or build status, hours until event or offer window ends
◆ Context data — channel (web, app, dealer, BDC), test-drive booked, trade-in valuation
◆ Logical operators — boosts, multipliers, eligibility & compliance rules
◆ Profile attributes — patient segment (virtual / specialty / employer), conditions, benefits tier
◆ Item attributes — care pathway, SLAs, hours until enrolment or appointment window ends
◆ Context data — channel (portal, app, telehealth), consent, formulary, network
◆ Logical operators — boosts, multipliers, clinical eligibility & policy rules

Method Three
AI Models
A machine learning model trained on real conversion events continuously scores and re-ranks items for each individual profile — in real time, automatically.
The hypothesis
"Not every customer converts for the same reason.
If we could learn what drives each individual, we could show the item most likely to resonate — not the one we think should work."
A streaming platform has 5 upgrade item variants — different value propositions, commitment levels, and price anchors. Every customer sees the upgrade prompt, but which variant they see is determined by a Personalized Optimization model trained on 6 months of real subscription events. The model doesn't follow rules a marketer wrote. It finds patterns humans wouldn't think to look for.
An airline publishes 7 ancillary and upgrade variants — bags, seats, priority, Wi‑Fi, bundles, and cabin upsells. Every traveller eligible for add-ons sees the same decision surface, but which offer ranks first comes from a Personalized Optimization model trained on months of purchase and hold events. It learns route, party size, and loyalty patterns — not a single global “best” ancillary.
A retailer runs dozens of concurrent promos across categories — delivery, loyalty, financing, bundles, and clearance. Every eligible shopper hits the same decision surface at checkout, but which offer ranks first comes from a Personalized Optimization model trained on purchase, browse, and abandonment events. It learns segment, basket, and channel — not one global “best” coupon.
A bank markets many product variants — cards, lending, savings, wealth, and insurance. Every eligible customer may see the next-best offer in-app, but which product ranks first comes from a Personalized Optimization model trained on applications, holds, and servicing events. It learns intent, risk band, and channel — not one global “best” rate sheet.
A telco publishes many plan and add-on variants — mobile tiers, fibre speeds, SMB packs, roaming, and devices. Every eligible subscriber may hit the same upgrade surface, but which offer ranks first comes from a Personalized Optimization model trained on usage, recharge, care, and install events. It learns segment, line mix, and channel — not one global “best” tariff.
An OEM and dealer network run many programme variants — PCP events, EV charger bundles, service plans, fleet terms, and approved used. Every eligible contact may see the same next-best-offer slot, but which programme ranks first comes from a Personalized Optimization model trained on configure, test-drive, handover, and workshop events. It learns journey stage, stock, and channel — not one global “best” incentive.
A health system and plan publish many care programme variants — virtual primary, urgent video, specialty expedited intake, pharmacy bundles, chronic coaching, mental health, and employer screening. Every eligible member may hit the same next-best-programme slot, but which offer ranks first comes from a Personalized Optimization model trained on booking, refill, risk screen, and benefits events. It learns care path, urgency, and channel — not one global “best” brochure.

Auto-Optimization
Optimizes for overall item performance. Surfaces whichever item variant drives the best conversion rate across the entire audience — same winner for everyone.
In our streaming example: the model would find that "30 Days Free" converts best overall and show it to everyone — ignoring that high-intent users like Maya would have converted on a higher-value plan.
In our airline example: auto-optimization might push the same checked-bag deal to everyone — even when Gold travellers like Alex respond better to cabin upgrades and families need row-together first.
In our retail example: auto-optimization might push the same sitewide coupon to everyone — even when Platinum shoppers like Jordan respond better to member electrics and households need bulk value first.
In our FSI example: auto-optimization might push the same generic savings rate to everyone — even when private-banking clients like Elena need wealth propositions first and SMB owners like Miguel need business bundles.
In our telco example: auto-optimization might push the same handset discount to everyone — even when mobile-heavy subscribers like Priya need roaming and unlimited first and fibre-led households like Dan need mesh and gigabit before mobile upsells.
In our automotive example: auto-optimization might push the same generic cash discount to everyone — even when showroom buyers like Hannah need stock-led PCP first, EV intenders like Omar need charger bundles, and fleet buyers like Claire need multi-vehicle care.
In our healthcare example: auto-optimization might push the same generic wellness tip to everyone — even when virtual-first members like Morgan need same-day video first, acute presenters like James need expedited specialty intake, and benefits admins like Taylor need employer screening kits.

Personalized Optimization
Predicts the best item per individual. Each profile gets a different score — the model learns from that person's own behavioural signals and similar users who converted.
In our streaming example: Maya sees the premium annual plan, Marcus sees the low-risk monthly trial, the Garcias see the family bundle — each ranked #1 because the model predicts it converts them.
In our airline example: Alex sees cabin upgrade first, Sam sees bags and seat selection, the Nguyens see family row + bags — each ranked #1 for that trip because the model predicts attachment for that traveller.
In our retail example: Jordan sees member electrics first, Riley sees value aisle and BNPL, the Parkers see kids bundles — each ranked #1 because the model predicts conversion for that shopper.
In our FSI example: Elena sees wealth advisory first, Chris sees cashback and balance transfer, Miguel sees business banking — each ranked #1 because the model predicts uptake for that customer.
In our telco example: Priya sees unlimited and roaming first, Dan sees fibre and mesh first, Renee sees business multi-line and SD‑WAN first — each ranked #1 because the model predicts attach for that subscriber.
In our automotive example: Hannah sees showroom PCP and part-exchange first, Omar sees EV bundle and charger install first, Claire sees fleet care and contract hire first — each ranked #1 because the model predicts conversion for that buyer.
In our healthcare example: Morgan sees virtual primary and pharmacy bundles first, James sees expedited cardiology and imaging first, Taylor sees employer screening and team challenges first — each ranked #1 because the model predicts the next completed care step for that member.
