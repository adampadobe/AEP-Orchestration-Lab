⚙ Sandbox & streaming connection Configure the streaming connection for this sandbox — required before sending consent updates. Fields are cleared when you switch sandboxes.

Before you can use the Consent Manager on a new sandbox:

  1. Create a Schema — In AEP, go to Schemas and create a new schema using the XDM Individual Profile class. Add these four field groups:
    • Profile Core v2 — custom tenant identity fields (includes email)
    • Preference Details — preferred language and communication preferences
    • Consent and Preference Details — marketing consent flags (email, SMS, push, call, postal, etc.)
    • Demographic Details — person name and demographic attributes
    Set the primary identity to _{tenant}.identification.core.email with namespace Email. Copy the Schema $id.
  2. Create a Dataset — In AEP, go to Datasets and create a new dataset from your schema. Enable it for Real-Time Customer Profile if you want consent updates to appear on the profile. Copy the Dataset ID.
  3. Create an HTTP API Dataflow — In AEP, go to Sources → Streaming → HTTP API and create a new streaming connection pointing at your dataset. Copy the Collection URL and Flow ID.
  4. Enter the details below — Paste Schema $id, Dataset ID, Flow ID, and Collection URL into the fields, then click Save connection. The connection is stored per sandbox in Firebase so you only need to do this once.

If this sandbox already has a saved connection, click Load from Firebase to populate the fields.

HTTP API streaming (Sources → Streaming → HTTP API)

Paste the flow ID from your HTTP API dataflow, or use Fetch URL & Flow ID from AEP to look it up by name. Then click Save connection.

Load from Firebase pulls any previously saved connection for this sandbox. Save connection stores all fields above to Firebase for this sandbox — you only need to do this once per sandbox.
