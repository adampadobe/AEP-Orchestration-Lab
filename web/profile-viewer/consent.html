View and update consent preferences

Query a profile by email and manage marketing, channel, and personalization consents in one flow.

🔍 Step 1: Query customer profile

Pick an identity namespace and enter the matching value (email, ECID, CRM ID, etc.). Experience Platform returns the profile; consent updates always stream using the profile’s email as the primary identity.

⚙ Sandbox & streaming connection Open to prepare the sandbox, HTTP flow, and connection — required before consent updates.

Run steps 1 → 4 in order for this sandbox. Do not enable this schema or dataset for Real-Time Customer Profile in the AEP UI — enabling the schema adds an irreversible union tag; this app never adds it. Step 1 creates schema shell AEP Profile Viewer - Consent - Schema (Profile class only, not Profile service). Step 2 attaches consent field groups + Profile Core v2 and primary identity on _{tenant}.identification.core.email. Step 3 creates dataset AEP Profile Viewer - Consent - Dataset (keep dataset out of Profile). Step 4: HTTP API dataflow AEP Profile Viewer - Consent - Dataflow; paste collection URL and Flow ID below.

HTTP API streaming (Sources → Streaming → HTTP API)

Only field you edit: paste the flow ID after you create the HTTP endpoint, then Fetch URL & Flow ID from AEP to load the collection URL. Other values are set by Prepare steps, status, or Fetch.

Save connection stores URL, flow ID, and IDs to Firebase for this sandbox. Clear Flow ID only if you need lookup by the default dataflow name instead of by ID.

✅ Step 2: Consent preferences

Choose Yes, No, or N/A for each item. Email and Push columns map to channel consent where applicable; merged rows apply one choice to both.

How we use your data

Core consent flags for this profile. Each row uses one choice across Email and Push.

Data collection Allow Experience Platform to collect profile and experience data for this identity.
Data sharing Allow sharing of this profile with Adobe solutions and connected destinations per your policies.
Content personalization Use profile attributes to tailor content and offers you see across channels.

Marketing communications

Email marketing (A-specific) Brand- or campaign-specific email consent tied to the same channel.

🚀 Step 3: Save changes to Adobe Experience Platform

Sends consent data to the DCS collection URL and flow you configured under Sandbox & streaming connection, using the selected sandbox and tenant key (e.g. _demoemea). The payload uses the profile email from Step 1 as the primary identity in identityMap and identification.core.email (ECID is included only when the loaded profile has one). Ensure URL, Flow ID, and (if envelope mode is on) dataset and schema IDs are saved before updating.
