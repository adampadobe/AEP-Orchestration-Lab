◆ APS payload

Same idea as Firebase Realtime Database: paste in Paste JSON, use Tree to browse like the RTDB tree view, then Parse & fill form or switch to Form to edit fields.

Paste the full unitary body, or only requestPayload / aps. Use Beautify, then Parse & fill form — we open the Form tab with fields filled (like switching from JSON to Tree on Firebase). Use the Execution & recipient buttons above to inject timestamp, requestId, and other values into the JSON.

Smallest useful paste is the aps object — see example below. Ctrl/⌘ + Enter runs Parse & fill form.

Pasting valid JSON auto-formats like the Realtime DB editor. Each Preview/Send still generates a new requestId unless you rely on the server payload builder only.

Example (Postman / AJO)

You can paste the full JSON below, or only the aps object. requestId is optional — each Preview/Send can generate a new UUID. Unquoted placeholders (e.g. "timestamp": {{$timestamp}}) are auto-quoted for parsing.

    "campaignId": "{{campaignID}}",

            "type": "aep",
            "userId": "{{ECID}}",
            "namespace": "ECID",
            "context": {
                "requestPayload": {
                    "aps": {
                        "content-available": 1,
                        "timestamp": 1735689600,
                        "event": "start",
                        "content-state": {
                            "boardingStatus": "Check-in Complete",
                            "statusMessage": "Proceed to security",
                            "timeStatus": "On time",
                            "dwellTimeMessage": "Explore KSIA's dining & shopping"
                        },
                        "attributes-type": "KSIAAirportAttributes",
                        "attributes": {
                            "flightNumber": "RX 123",
                            "airline": "Riyadh Air",
                            "departureAirport": "RUH",

Browse like Firebase Realtime Database → Tree. Click a value to edit it; changes update the JSON in Paste JSON. Expand all opens every branch and grows the tree to use the viewport.

Edit value

Scalars

timestamp (Unix seconds) is set when you click Preview or Send — not taken from pasted JSON.

event (start / update / end) is set in Execution & recipient above — same value is used when you Preview or Send.

Payload fields

Each block has Form (key/value rows) and JSON. Paste a fragment or full aps into any JSON field to import — or use the Paste JSON tab for the whole body.

content-state Short status lines shown on the Live Activity (e.g. boarding, gate).

attributes Structured data for your attributes-type (nested objects use a small JSON box).

alert Notification title and body text.
