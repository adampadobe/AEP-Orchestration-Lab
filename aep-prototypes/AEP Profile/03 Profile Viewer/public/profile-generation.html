🧬 Generic profile (industry-less)


      
      

        
          
            ⚙
            Sandbox & streaming connection
          
          Per-sandbox setup. Run the four steps once per sandbox, then create the HTTP API dataflow in AEP and click Fetch URL & Flow ID.
        
        

          

            
Before you can generate or update generic profiles in a new sandbox:

            
    
              
  1. Create schema — Creates an empty Profile schema named AEP Lab - Generic Profile - Schema.
    2. 
              
  2. 
                Attach field groups — Adds Profile Core v2, Demographic Details, Personal Contact Details, Loyalty Details, Consent and Preference Details (OOTB consents.marketing.*, including the preferred-channel enum at consents.marketing.preferred), and Preference Details (preferences.*, e.g. preferred language) to AEP Lab - Generic Profile - Schema, then sets the primary Email identity descriptor on _{tenant}.identification.core.email. The wizard does not enable the schema for Real-Time Customer Profile at this point — that is a deliberate, manual step (see note below).
              
    3. 
              
  3. Create dataset — Creates AEP Lab - Generic Profile - Dataset.
    4. 
              
  4. Create an HTTP API Dataflow — In AEP, go to Sources → Streaming → HTTP API and create a new streaming connection pointing at your dataset. Name the dataflow AEP Lab - Generic Profile - Dataflow (the value pre-populated in the Dataflow name (used for lookup when Flow ID is empty) field below) so Fetch URL & Flow ID from AEP can resolve the Flow ID and Collection URL automatically — or copy the Flow ID manually if you used a different name.
    5. 
            

            

              ⚠ Manual step (in AEP UI, after the wizard): When you are satisfied with the schema's field groups, open AEP Lab - Generic Profile - Schema in the AEP UI and toggle Profile ON yourself. The wizard intentionally never sets this flag because enabling a schema for Profile is a one-way action in AEP — once enabled you cannot add or remove field groups, and the schema becomes very difficult to retire. Only toggle it on after you confirm step 2 attached every field group you need.
