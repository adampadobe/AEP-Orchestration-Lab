Decisioning lab setup

Configure once per sandbox (saved to Firebase — sign in anonymously). Create a datastream in Data Collection / DSN, paste its ID plus your Adobe Tags Launch script URL below, then define placement fragments that match Journey Optimizer code-based surfaces (web://…#fragment).

1. Create a datastream in DSN

Datastreams are created outside this lab. Use one of the Demo Sandbox Network portals:

  • https://livedemos.adobe.com/ — live-demos DSN
  • https://dsn.adobe.com/ — standard DSN

Inside DSN, create (or pick) a datastream wired to your sandbox with the services you need (AEP, AJO, Personalization). Copy its Datastream ID and the Launch embed-script URL from the Web SDK extension — you'll paste both below.

3. Placement fragments (AJO surface hash)

Each row is a preview region. Key becomes the DOM id cd-edge-… and must stay unique. Fragment is the hash in web://host/path#fragment and must match your Journey Optimizer code-based channel for this page.

Before a fragment can return personalisation

Every placement / fragment you add here also needs a matching code-based channel configuration in Adobe Journey Optimizer. Create the channel config with surface web://<your-host>/<path>#<fragment>, then author a code-based experience against it. Until the channel config exists, the edge will respond with zero propositions for that fragment.

Docs: Create a code-based channel configuration · Code-based experiences overview · Create a code-based experience

Placements are persisted by the Save to Firebase button in Step 2.

4. Surface styling

AJO delivers the content (title, image, text, link) as JSON. The styling — layout, alignment, colours — is yours. Pick a surface and adjust. You can also hide returned slots (e.g. the ribbon CTA) and set a preview box height (e.g. 100px) for that mount. Changes apply instantly and save per sandbox + surface in Firebase.
