Reporting

Event volumes from the AEP lakehouse (Query Service), grouped by eventType. By default all events in the range are included; set AEP_REPORTING_JOURNEY_SQL_PREDICATE on the server to restrict to journey-linked rows when your dataset has AJO fields.

📊 Events timeline

Daily counts of experience events grouped by eventType (all rows in the date range by default). Data is loaded via AEP Query Service on your events dataset (AEP_QUERY_EVENTS_DATASET_ID). To limit to AJO journey-linked rows only, set AEP_REPORTING_JOURNEY_SQL_PREDICATE to a fragment that references columns your lakehouse actually has (for example _experience.customerjourneymanagement.messageexecution.journeyVersionID when present). Customer Journey Analytics (CJA) is a separate product—this view does not call CJA APIs; for Workspace-style reports use Adobe’s CJA tools or connect CJA APIs in addition.
